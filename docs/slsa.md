# SLSA Provenance Policy — Configra

**Version:** 0.1.0
**Date:** 2026-06-19
**Status:** ACTIVE (T10.1 remediation, Gate 3)
**Target level:** SLSA Build Level 3

---

## Scope

This document defines the SLSA (Supply-chain Levels for Software Artifacts)
provenance policy for **Configra** workspace releases. It applies to:

- All crates in the `crates/` workspace
- The TypeScript edge layer in `typescript/`
- Container images published from this repo (planned)

## Source

SLSA Framework v1.0: <https://slsa.dev>
SLSA Build Level 3: <https://slsa.dev/build/v1.0/levels#build-l3>

## Build provenance

Every release artifact (crate tarball, npm tarball, container image) must
have provenance in the form of:

- **Signed attestation** (in-toto) → `release-attestation.yml`
- **SLSA provenance** (slsa-github-generator) → `slsa-provenance.yml`

## Requirements (Build Level 3)

| Requirement | Implementation |
|---|---|
| Provenance generated by hosted build platform | GitHub Actions runners (hardened) |
| Provenance signed by build platform | `slsa-framework/slsa-github-generator/.github/workflows/generator_container_slsa3.yml` |
| Build can be reproduced from provenance | `cargo build --workspace --locked` |
| Build script is isolated | GitHub-hosted runner, no network beyond allowlist |
| Provenance is non-forgeable | Signed by GitHub OIDC token + cosign keyless |
| Provenance is available to consumers | Attached to release artifacts |

## Workflows

### `.github/workflows/release-attestation.yml`

Triggered on `v*` tag push. Steps:

1. `cargo build --workspace --release`
2. `cargo package` for each workspace member
3. Generate in-toto attestation linking source → artifact
4. Sign with cosign keyless (GitHub OIDC)
5. Attach `.att` file to GitHub Release

### `.github/workflows/slsa-provenance.yml`

Triggered on `v*` tag push. Steps:

1. `slsa-framework/slsa-github-generator` (Generator v2.x)
2. Output: `.intoto.jsonl` provenance file
3. Attach to GitHub Release

## Verification

Consumers can verify provenance:

```bash
# Install slsa-verifier
go install github.com/slsa-framework/slsa-verifier/v2/cli@latest

# Verify a release artifact
slsa-verifier verify-artifact <artifact> \
  --provenance-path <artifact>.intoto.jsonl \
  --source-uri github.com/KooshaPari/Configra
```

## Key management

- **Signing:** cosign keyless (GitHub OIDC, Fulcio)
- **Verification:** Rekor transparency log (public)
- **Key rotation:** automatic (keyless = per-attestation ephemeral key)
